Using Adobe InDesign

What is Prototyping for Web?

Prototyping for web is a non-programming interactive design layout for user interface testing.

Essentially it is a mock up or wireframe that is an interactive demo of your site early in the pre-development phase without using any programming or code.

Why we should prototype

Originally InDesign was built to produce portfolio and interactive magazines with a flipbook element using Flash. As technology has advanced, so haven't the uses for Adobe Indd. Flash is now a thing of the past, but wireframe prototyping became a new interest within the program because it had the same conceptual design as an interactive magazine without having to use a plug-in.

Many designers who create eBooks love using InDesign for its seamless structure and file implementation across all the designer develpment tools (ie: Photoshop, Illustrator, etc.).

Static designs, wireframes and site maps are great when starting the pre-development stage for designing your website, but they don't illustrate the level of detail your site can acheive when portraying to the end user or client. Hence the use of real content in our prototypes helps the end user/client evaluate the site's design in real time prior to development.
